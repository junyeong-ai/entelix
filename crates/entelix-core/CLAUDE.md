# entelix-core

DAG root of the workspace. Depends on **no** other entelix crate. Adding any `entelix-*` to `Cargo.toml` here breaks the workspace — wire the dependency the other way around.

## Surface

- **IR** (`ir::`) — provider-neutral payloads (`ModelRequest`, `ModelResponse`, `Message`, `ContentPart`, `ToolUse`, `ToolResult`, `StopReason`, `ModelWarning`, `ProviderExtensions`, `ProviderEchoSnapshot`). `ModelRequest::continue_turn(prior_response, next_message)` advances a request to the next conversational turn — appends the prior assistant turn, chains `provider_echoes` into `continued_from`, pushes the new message (state-transition verb per `.claude/rules/naming.md`).
- **Codec** (`codecs::`) — `Codec` trait + one impl per vendor wire format (Anthropic Messages, OpenAI Chat, OpenAI Responses, Gemini, Bedrock Converse). Stateless, totalable: `encode` is total, `decode` is panic-free on arbitrary bytes.
- **Transport** (`transports::`) — `Transport` trait + `DirectTransport`. Stateful (auth + connection pool). Cloud-routed transports (Bedrock/Vertex/Foundry) live in `entelix-cloud`.
- **Tool** (`tools::`) — `Tool<D = ()>` trait, `ToolRegistry<D = ()>`, `ToolMetadata`, `ToolEffect`, `RetryHint`. In-crate tool-side layers: `RetryToolLayer` (`NAME = "tool_retry"` — per-tool `RetryHint`-driven retry, distinct from `transports::RetryLayer`'s policy-driven transport retry) and `ScopedToolLayer` (`NAME = "tool_scope"` — operator-supplied `ToolDispatchScope` wraps every dispatch future for ambient task-locals / tenant-RLS). Single `execute(input, ctx: &AgentContext<D>) -> Result<Value>` method (invariant 4). `D` is operator-supplied typed deps (default `()` for the deps-less zero-cost path); the layer ecosystem consumes the D-free `ToolInvocation` so existing layers compile unchanged regardless of `D` (invariant 19). `ToolRegistry::layer<L: NamedLayer + ...>` composes a tool-side layer, `ToolRegistry::layer_named(name, layer)` is the `WithName` sugar variant for external middleware, and `ToolRegistry::layer_names() -> &[&'static str]` surfaces the composed stack. No back-channels.
- **Tool progress** (`tools::progress::`) — `ToolProgressSink` trait + `ToolProgressSinkHandle` (operator-attached via `ExecutionContext::with_tool_progress_sink`) + `CurrentToolInvocation` (per-dispatch identity marker the layer ecosystem stamps on entry) + `ToolProgress { phase, status, dispatch_elapsed_ms, metadata }`. Tools emit phase transitions via `ctx.record_phase(phase, status)` / `record_phase_with(phase, status, metadata)`. Fire-and-forget by contract — sink failures stay inside the sink (`AuditSink`-style, invariant 18) and never propagate into `Tool::execute`. Silent no-op when no sink or no dispatch marker — library tools can call without depending on a particular runtime.
- **Auth** (`auth::`) — `CredentialProvider` + `ApiKeyProvider`, `BearerProvider`, `Cached*`, `Chained*`. Tokens never leave this module via context (invariant 10).
- **Service spine** (`service::`) — `ModelInvocation` / `ToolInvocation` / `StreamingModelInvocation` `tower::Service` types. Cross-cutting concerns plug in as `tower::Layer<S>`. `NamedLayer` trait (sync `fn layer_name(&self) -> &'static str`) is required on every layer composed via `ChatModel::layer` / `ToolRegistry::layer`; the compose sites capture `layer.layer_name()` into a `Vec<&'static str>` surfaced via `*::layer_names()` for boot-time wiring diagnostics. External `tower::Layer` middleware participates through `WithName::new(name, inner)` — the wrapper is transparent at the `tower::Layer` boundary.
- **`ExecutionContext`** (`context::`) — request-scope carrier. Carries `cancellation`, `deadline`, `thread_id`, `tenant_id` (mandatory — invariant 11), `run_id` / `parent_run_id`, `idempotency_key`, `run_budget`, `audit_sink`, `tool_progress_sink`, plus a type-keyed `extensions` slot. Single-slot fields use `with_*` setters; the extensions slot uses `add_extension` (per naming taxonomy).
- **`AgentContext<D>`** (`agent_context::`) — typed-Deps carrier. Wraps `ExecutionContext` plus an operator-supplied typed handle `D` (defaults to `()`). Tool dispatch threads `&AgentContext<D>` to leaf execution; the `D`-free `ToolInvocation` rides through every layer so the ecosystem stays generic-free (invariant 19).
- **`Extensions`** (`extensions::`) — type-keyed `Arc<HashMap<TypeId, …>>`, copy-on-write. Mirrors `http::Extensions`. **Never stash credentials here** — invariant 10 alignment.
- **Chat** (`chat::`) — `ChatModel` + `ChatModelConfig`. `complete` (text), `complete_full` (full `ModelResponse`), `complete_typed::<O>` (typed structured output via `LlmFacingSchema::strip` + `serde_json` deserialization), `complete_typed_validated::<O>(., validator)` (the typed path with an `OutputValidator<O>` semantic check). `with_validation_retries(n)` enables the unified retry loop that catches both schema-mismatch and validator failures through `Error::ModelRetry` (invariant 20). `layer<L: NamedLayer + ...>(layer)` composes a layer onto both one-shot and streaming spines and records `layer.layer_name()`; `layer_named(name, layer)` is sugar for `layer(WithName::new(name, layer))` so external `tower::Layer` middleware composes without implementing `NamedLayer` manually. `layer_names() -> &[&'static str]` returns the composed stack in insertion order (last element = outermost).
- **`OutputValidator<O>`** (`output_validator::`) — post-decode semantic validator. `Fn(&O) -> Result<()>` blanket impl makes the common case a closure; validators returning `Err(Error::ModelRetry { hint,. })` route through the `ChatModel` retry loop with the hint reflected to the model as a corrective `User` message.
- **Per-call overrides** (`overrides::`) — `RunOverrides` (loop-level: `model` / `system_prompt` / `max_iterations`) and `RequestOverrides` (request-level: `temperature` / `top_p` / `top_k` / `max_tokens` / `stop_sequences` / `reasoning_effort` / `tool_choice` / `response_format` / `parallel_tool_calls`). Both flow through `ExecutionContext::add_extension`; `chat::apply_overrides` merges them onto the outgoing `ModelRequest`. Split is by *who chooses to override* (agent-loop vs request-shape), not by where the field lives in IR.
- **Interruption** (`interruption::`) — single HITL primitive. `InterruptionKind::{Custom, ApprovalPending { tool_use_id }, ScheduledPause { phase, node }}` typed enum. `interrupt(payload)` / `interrupt_with(kind, payload)` raise `Error::Interrupted { kind, payload }` from inside any `Tool::execute`, graph node, or `tower::Layer` — one shape, one resume path (`CompiledGraph::resume_with(Command)`).
- **Time** (`time::`) — `Clock` trait + `SystemClock` (delegates to `tokio::time::Instant`). DAG-root location so rate limiters, retry backoff, TTL pruning all consume one time abstraction without depending on `entelix-policy`.
- **Run budget** (`run_budget::`) — `RunBudget` axis cap (requests, input/output/total tokens, tool_calls, USD cost). `UsageLimitBreach` typed enum pairs each axis with its magnitude shape (`u64` count or `Decimal` USD) — `Error::UsageLimitExceeded(breach)` carries one variant. Sub-agent fan-out shares `Arc<RunBudgetState>`. `UsageSnapshot` exposes the frozen counters on `AgentRunResult`.
- **Tokens** (`tokens::`) — `TokenCounter` trait + `ByteCountTokenCounter` (conservative over-count fallback) + `TokenCounterRegistry` + `Resolution { Matched, Fallback }`. The registry routes `(provider, model)` pairs to a vendor-accurate counter via longest-prefix matching; `resolve` returns a typed `Resolution` so production paths branch on the miss instead of silently absorbing the fallback (invariant 15). Operator-extensible via `register(provider, model_prefix, counter)`. Companion crates `entelix-tokenizer-tiktoken` / `entelix-tokenizer-hf` ship concrete `TokenCounter` impls; the facade's `default_token_counter_registry()` pre-populates every known OpenAI BPE family — including `gpt-4.1` / `gpt-4.5` carve-outs that route to `o200k_base` against the `gpt-4` cl100k catchall.
- **Error** (`error::`) — `Error` (top-level public API), `ProviderErrorKind` (`Network` / `Tls` / `Dns` / `Http(u16)` — typed transport-failure category), `ErrorClass` (`Client` / `Server` — HTTP-family responsibility split). `Error::provider_http(status, …)` coerces statuses outside `400..=599` (zero, 1xx/2xx/3xx, 6xx+) to `ProviderErrorKind::Network` so a non-terminal status never surfaces under a plausible-looking `upstream_error` (invariant 15). `Error::envelope() -> ErrorEnvelope` is the single canonical inspector — a `Copy` value bundling `wire_code: &'static str` (patch-version-stable `"rate_limited"` / `"upstream_unauthorized"` / `"auth_failed"` / `"transport_failure"` / …), `wire_class: ErrorClass`, `retry_after_secs: Option<u64>` (vendor `Retry-After` from `Error::Provider::retry_after`), and `provider_status: Option<u16>` (raw HTTP status). Derives `serde::Serialize` (lowercase `ErrorClass`, `Option`s skipped when `None`) so sinks forward to SSE / JSON in one step. Retry-eligible classification stays orthogonal — vendor `Retry-After` rides on `Error::Provider::retry_after` (invariant 17).

## Crate-local rules

- New codec / transport / tool: implement the trait, add the `*Codec` / `*Transport` / `*Tool` suffix, do not add a method to the trait — extend metadata struct or add a sibling trait instead.
- New IR field: bump `ProviderExtensions` for vendor-specific knobs; only add to `ModelRequest`/`Response` itself if every codec carries it natively.
- Lossy codec encoding: emit `ModelWarning::LossyEncode { codec, dropped_field }`. Silent drop fails review (invariant 6).
- `tests/codec_robustness_proptest.rs` is load-bearing — run after any codec change.

## Forbidden

- `std::fs`, `std::process`, `tokio::fs`, `tokio::process`, `landlock`, `seatbelt` (invariant 9).
- Embedding `CredentialProvider` in `ExecutionContext` (invariant 10).
- Returning `anyhow::Error` from public API.
- Trait methods returning vendor-shaped JSON (invariant 5).
