# entelix-tools

Built-in `entelix_core::tools::Tool` impls. Hard boundary against invariant 9: any shell / filesystem behaviour delegates through `Sandbox`.

## Surface

- **`HttpFetchTool`** + `HostAllowlist` — HTTP fetch with **mandatory** host allowlist (SSRF defense). Builder without any allowed host produces a tool that refuses every URL — fail-closed by construction. Three-layer SSRF defense: (1) allowlist check on URL host, (2) DNS resolution check via `SsrfSafeDnsResolver` against RFC1918 / loopback / link-local, (3) redirect cap + response-size cap.
- **`CalculatorTool`** — recursive-descent arithmetic over `f64`. No `eval`, no shell-out.
- **`SearchProvider` trait** + `SearchTool` — adapter for external search APIs (Brave / Tavily / Perplexity / …). Concrete providers are 1.1 companions (trait-only here per ADR-0008 policy).
- **`SchemaTool`** + `SchemaToolAdapter` + `SchemaToolExt` — typed-I/O sibling of `Tool` for tool authors. `Input: DeserializeOwned + JsonSchema`, `Output: Serialize`, schema autogenerated via `schemars::schema_for!(T::Input)` and cached in `Arc<ToolMetadata>`. `.into_adapter()` exposes any `SchemaTool` through the erased `Tool` trait.
- **`sandboxed`** — `SandboxedShellTool`, `SandboxedCodeTool`, `SandboxedReadFileTool`, `SandboxedWriteFileTool`, `SandboxedListDirTool`. All hold `Arc<dyn Sandbox>` and never touch `std::process` or `std::fs` directly.
- **`skills`** — `InMemorySkill`, `SandboxSkill`, plus the LLM-facing tools (`ListSkillsTool`, `ActivateSkillTool`, `ReadSkillResourceTool`) for the progressive-disclosure pattern (ADR-0027).
- **`memory`** — `MemoryGetTool`, `MemoryPutTool`, `MemorySearchTool` over the `entelix_memory` traits.

## Crate-local rules

- **Invariant 9 is a hard wall here.** This crate is the *only* place in the workspace where shell / filesystem semantics surface, and even here the actual syscalls go through the `Sandbox` trait. Adding a `tokio::fs::read` to a sandboxed tool defeats the whole boundary.
- **`HttpFetchTool` allowlist is opt-in, fail-closed.** Empty allowlist refuses every URL — never "default to allow all" for convenience.
- **`SchemaTool` authors do not touch `Tool` directly.** The adapter machinery wires metadata; trait authors override `effect` / `version` / `retry_hint` / `output_schema` instead.
- New typed tool: implement `SchemaTool`, derive `Deserialize + JsonSchema` on `Input`, `Serialize` on `Output`, register via `.into_adapter()`.
- Tools that take long: poll `ctx.is_cancelled()` in inner loops.

## Forbidden

- `std::fs`, `tokio::fs`, `std::process`, `tokio::process` anywhere in this crate (invariant 9).
- `HttpFetchTool` with permissive default (any allowlist that admits a URL by default).
- A `Tool` impl reading credentials from `ExecutionContext` (invariant 10).
