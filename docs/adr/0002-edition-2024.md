# ADR 0002 — Rust 1.94 + Edition 2024

**Status**: Accepted
**Date**: 2026-04-26

## Context

entelix starts new in 2026-04. Rust 1.94 (released 2026-03-02) is the current stable. Edition 2024 has been stable since Rust 1.85 (Feb 2025). Resolver v3 ships with edition 2024.

The async-fn-in-trait stabilization (RFC 3185, stabilized 1.75) and edition-2024 trait async fn lifetimes (RFC 3683 mostly resolved by 1.94) reduce or eliminate the need for `async-trait` in many cases.

## Decision

- **Toolchain**: pin to `1.94.0` via `rust-toolchain.toml`.
- **Edition**: `2024` for every member crate.
- **Resolver**: `"3"` (workspace-level).
- **MSRV policy**: track stable - 0; bump MSRV is a minor version event.

`async-trait` is still listed in `[workspace.dependencies]` because:
- Object-safety in some Tool/Codec dispatch sites still benefits from it.
- Boxing the future to keep a stable ABI when sending across thread boundaries is sometimes preferable.
We use **native async fn in trait** by default, fall back to `#[async_trait]` only when the dispatch is `dyn Trait` and we want a uniform `BoxFuture` return.

## Consequences

✅ Modern syntax (let-else, GATs, async closures partial).
✅ Resolver v3: per-target feature unification fixes the historical "everyone gets aws-lc-rs" footgun.
✅ Edition 2024 future-incompat lints catch issues at compile time.
✅ Free of `async-trait` in 80%+ of trait definitions — cleaner rustdoc.
❌ Users on older toolchains cannot compile entelix. Acceptable: this SDK is for greenfield production work.
❌ Some dependencies still ship 2021-edition crates; mixed editions per crate are fine, but interop care (e.g., `&dyn Iterator` lifetime elision rules differ).

## Implementation notes

- Set `[workspace.package].rust-version = "1.94"` and inherit in each member.
- CI matrix tests `1.94` only (no MSRV-back testing) until first 1.0 release.
- `rust-toolchain.toml` pins exact version with `rustfmt`, `clippy`, `rust-src` components.

## Open question

Native `async fn` in trait still does not allow specifying `Send` bounds without per-method workarounds. If we hit ergonomic issues with sub-agent spawning across threads, we may localize `async-trait` use. Track in Phase 1.
